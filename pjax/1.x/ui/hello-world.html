几个问题：

1） Admui 不支持内联的 js。
2） Admui 是通过 Pjax 来加载页面的，其中的 js 文件都是异步加载的，并不会按照 js 文件顺序进行运行。因此，对于引用插件有依赖关系的，可以通过data-name和data-deps来写入依赖关系，Admui 会根据 data 属性对依赖的 js 进行同步加载，如：

3） 为什么要通过 window.Content 对象这样的形式来写js呢？因为通过window.Content对象写，Admui 会在新建标签页时保证添加的 js 文件顺序运行。当然您也可以不使用该对象，此时您就需要在引入该 js 文件时增加对其他文件的依赖关系。
4） 绑定事件时，尽可能的绑定在 $('#admui-pageContent') 对象上。在新打开标签页时，我们会解绑该对象上的事件，避免造成污染。

相对路径与绝对路径

通常情况下，相对路径一般使用较多，但是在 Admui 中，我们提倡使用绝对路径，这样可以避免很多问题。
